GeQE (Geo-Query by Example)

A User Walkthrough

GeQE (Geo-Query by Example) is a tool that aims to leverage geotagged social media data to identify events, based upon models built from similar events.  For example, given a set of data containing social media feeds from one city, the user is interested in posts from hospitals in the area.  The user can build a model from the posts in that city, and then apply it to datasets from other cities to pull hospital related posts.  The available datasets are from Boston, New York City, and Washington DC.  The jobs that have already been processed and available for analysis are related to airports, hospitals, and parks.  Examples referenced below include models built on location and events.

GEQE – Displaying Data from a Job

Once a job has been submitted and successfully run, the user can view the data from that job.

Underneath the map, in the JOBS tab, select a job, and then click on the ‘view job data’ button (eye icon).

In this example, a location based job related to hospitals in the Cleveland, OH area is used.  

The map is now populated with clusters of social media posts from the job file.  Each cluster is marked with a pin and the area is defined by a shaded polygon, representing the boundary of tweets selected by the analytics.  In this example, the clusters center on hospitals in the area.

The user can zoom in on the map using the plus/minus buttons in the lower right hand corner of the map, or by simply using the mouse scroll wheel.  

A bar graph in the upper right hand corner displays data on the scores and cluster sizes.

Left-click on a cluster pin and the word cloud beneath the graph displays the terms used in scoring.

In the bottom right hand corner is a list of terms used in scoring.

Clicking on a cluster pin will populate the right hand column of the screen with information about the tweets represented in the selected polygon.  Double clicking on a cluster pin will display markers for each tweet that define the cluster polygon.

Clicking on a data point marker will display information to the right of the map.

Note that top ranking terms are highlighted within the display.

The user may select an individual marker or use the left and right arrow keys in the information box on the right to view additional information for each tweet; user name, date/time, and text.  Words relevant to the scoring of the cluster are highlighted in orange.  

When searching through jobs based on time and date, the bar graph in the right hand column represents each date relative to the data set.  The orange bars represent the number of clusters for a given date and the blue line represents the number of posts.  Clicking on a bar adjusts the map zoom level to view the clusters for that date.  

 (Example below: two clusters of social media posts on a specific date related to the NY marathon).  Date selected in bar graph contains two clusters and 33 data points)

        

The user can select a cluster on the map to view the posts in the right hand column.

The Filter field can be used to narrow down the posts from the selected cluster.
